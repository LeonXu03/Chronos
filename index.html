Introduction

Welcome to the Chronos user manual, a comprehensive guide for working with the Chronos Discord bot.

What Does The Chronos Discord Bot Do?

Chronos was created to streamline the program delivery process. Chronos currently features a variety of functions, including taking attendance, scheduling messages, merging channels, and more. These features can be accessed by sending a text message featuring a “$” prefix followed by a command to a Discord text channel

Who Can Access The Chronos Discord Bot’s Commands?

Chronos’ commands are split into two categories: admin & participant. Discord users with the role names @HtCtW Team or @Tech Support or @How to Change the World Team are given admin privileges to use the Chronos Discord Bot.

Command AccessCommands Example
AdminALL$attendance $schedule $delete $reminders + participant commands + etc.
ParticipantLimited$zoom $background $program $help, $command, $commands

How Should I Setup Discord?

To maximize Chronos’ functionality, the following should be setup:

Categories:

  • Expert : Cohort # (for each cohort. Expert text channel rooms need to be setup here)

Text Channels:

  • #team-x-y (explained in commands)
  • #admin (a chat to type commands without actually pinging users)
  • #expert-and-expert-room channels depending on current Expert session

Roles:

  • @Team X Y (explained in commands)
  • @Tech Support (for admin role)
  • @HtCtW Team (for admin role)
  • @How to Change the World (for admin role)
  • @Expert Session (for live Expert session attendance)
  • A @Expert Name Room for each Expert expected to attend the program

For Expert Attendance:

Please setup categories following the format of “Expert : Cohort #” with Expert text channels in them.

For each expert text channel, each designated expert for that text channel should be given appropriate permissions by assigning that text channel with each expert’s corresponding roles ( following the format “@expert name room”).

Expert attendance is always based on the roles found in the server’s current expert text channels. Adjust the expert text channels and their roles as necessary throughout the program to reflect the current Expert session.

How Do I Read The Commands Section?

$command type

$command format (what format you need to follow for each Discord command)

The $command type is just the higher level of the command. There may be multiple commands under that higher level depending on the number of arguments passed.

The $command format is basically what you would actually type into Discord. Each Discord command argument, which look like [argument] in the commands below, must be separated by a space. Each argument is either encased in quotation marks or just typed directly into Discord (i.e. never enclosed in square brackets - they are just used below to visually separate arguments)

COMMANDS

Admin: $attendance, $schedule, $reminders, $delete, $merge, $set_zoom

Participant: $zoom, $background, $program

Miscellaneous: $help, $command, $commands

$attendance admin

$attendance

Shows attendance for @everyone in the server excluding bots (based on voice presence).

Example

$attendance @rolehere

Shows attendance for @rolehere in the server based on voice presence

Example

Special case:

$attendance @Expert Session

Shows attendance for Experts expected for the day. It looks for Experts based on the roles found in the expert text channels currently setup. Shows whether they are present, absent, or not registered on the server.

Example

$schedule admin

$schedule [“message”] [date] [time] [channel]

Schedules a reminder to be sent to a channel at a specific datetime. (more features)

Example

$schedule [“message”] [date] [time] [channel] [# of repeats] [time between repeats]

Schedules reminders to be sent to a specific channel at multiple datetimes as specified by the user. (more features)

Example

  • Proper formats for arguments
    ArgumentWhat To Type in Discord
    [”message”]“message” (as of 2022-04-25: cannot include quotation marks inside message → alternative is to use two ‘ characters)
    [date]YYYY-MM-DD
    [time]HH:MM
    [channel]#channel
    [# of repeats]#
    [time between repeats]#d/h/m (a number next to the letter d or h or m. d = days, h = hours, m = minutes. Ex: 3d or 15m)

  • #team-x-y and @Team X Y

    When scheduling a message, if you send a message to #team-x-y, this message will be sent to all team chats following the structure #team-x-y.

    IF the channel selected is #team-x-y AND the role @Team X Y is found in the “message”, the message will ping each team it is sent to.

    WARNING: If @Team X Y is found in the “message” but the message is NOT directed to #team-x-y, it will just ping @Team X Y (which is useless)

$merge admin

$merge #channel_1 → #channel_2

Merges channel_1 into channel_2. This works for #team-x-y and #open-innovation-table-# channels. Gives @channel_2 role permission to @channel_1 members (more). Also sends a closing message to the channel being closed and a welcome message to the channel inviting new members. Type $confirm after merge command to complete role transfer & send notifying messages to affected text channels.

Example

  • How are permissions shared?

    Open innovation tables: the @Open Innovation Table 2 role is given to all members with the role @Open Innovation 1

    Team chats: @Team X Y role of #channel 2 is given to all members with the role @Team X Y of #channel 1

    Important: no roles are deleted, you can delete them yourself on the “SUCCESSFUL COMMAND: $merge” Discord embed pop-up by clicking on the “Users given role”.
